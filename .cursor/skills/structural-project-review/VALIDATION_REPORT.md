# Structural Project Review Skill Validation Report

## 验证结论

当前 `.cursor/skills/structural-project-review/` 已经具备第一版稳定使用条件。

它可以稳定支撑以下工作：

1. 项目资料包完整性审查。
2. 文件命名、版本、重复和疑似过期文件识别。
3. 项目阶段识别。
4. 阶段准入/退出判断。
5. 结构风险识别。
6. DFM 问题提取和分类。
7. ECO/ECN 影响分析。
8. 测试、会议行动项和问题闭环检查。
9. 项目归档审查。
10. 高频问题和经验沉淀。
11. 平台交互、导出、权限、日志和规则迭代设计。

稳定性评级：`可进入试用验证`

不建议直接进入正式质量体系使用。原因是公司真实文件字典、审批角色、QMS 风险等级、ECO 流程和报告是否可作为正式质量记录仍需内部确认。

## 典型场景模拟

### 场景 1：上传结构方案资料包，要求判断是否可进入详细设计

模拟输入：

```text
03_结构方案/
├── 血压仪项目_结构方案_结构方案说明_RevA_20260603_评审版.pptx
├── 血压仪项目_结构方案_初版BOM_RevA_20260603_评审版.xlsx
└── 血压仪项目_结构方案_结构风险清单_RevA_20260603_评审版.xlsx
```

预期 Skill 行为：

1. 先执行文件结构和命名审查。
2. 判断阶段为 `结构方案`，置信度为中到高。
3. 发现缺少爆炸图或关键截面图。
4. 根据 `结构方案 -> 详细设计` Gate 输出未满足项。
5. 不直接建议进入详细设计，改为“条件通过/需补充关键截面图后确认”。

稳定性判断：通过。

### 场景 2：上传 DFM 报告，要求提取问题清单

模拟输入：

```text
05_DFM/
├── 血压仪项目_DFM_供应商DFM报告_RevB_20260605_供应商版.pptx
└── 血压仪项目_DFM_DFM会议纪要_RevA_20260606_内部版.docx
```

预期 Skill 行为：

1. 识别为 `DFM专项` 或自动触发 DFM 模块。
2. 输出 DFM 问题总数、高/中/低风险数量和高频问题类型。
3. 保留供应商原始意见，不把 AI 推断写成供应商结论。
4. 标记可能触发改图或 ECO 的问题。
5. 输出 DFM 会议讨论清单和供应商回复建议。

稳定性判断：通过。

### 场景 3：上传 ECO/ECN 资料，要求判断是否建议批准

模拟输入：

```text
08_ECO_ECN/
├── 血压仪项目_ECO_ECO申请单_RevA_20260610_待会签.docx
├── 血压仪项目_ECO_变更前后对比_RevA_20260610_评审版.pptx
├── 血压仪项目_ECO_变更影响分析_RevA_20260610_评审版.xlsx
└── 血压仪项目_ECO_验证计划_RevA_20260610_待确认.docx
```

预期 Skill 行为：

1. 识别为 `ECO专项`。
2. 输出变更摘要、变更原因、影响范围、验证建议、文件更新清单和会签建议。
3. 检查结构、模具、BOM、生产、检验、可靠性、法规注册、包装标签、库存和售后影响。
4. 将“是否建议批准”标记为 AI 辅助建议。
5. 明确 ECO 批准结论必须人工确认。

稳定性判断：通过。

### 场景 4：上传 DVT 测试报告，要求判断是否可进入 PVT

模拟输入：

```text
07_EVT_DVT_PVT/
├── 血压仪项目_DVT_跌落测试报告_RevA_20260615_已完成.pdf
├── 血压仪项目_DVT_DVT问题清单_RevA_20260615_评审版.xlsx
└── 血压仪项目_ECO_ECN_ECN通知单_RevA_20260616_已会签.docx
```

预期 Skill 行为：

1. 判断阶段为 `DVT` 或 `EVT/DVT/PVT`。
2. 检查 DVT 报告是否完整。
3. 检查结构相关失败项是否关闭。
4. 检查 ECO/ECN 是否完成会签和验证。
5. 若缺少复测通过证据，输出 `不建议进入 PVT / 需确认`。

稳定性判断：通过。

### 场景 5：上传最终归档资料，要求输出经验总结

模拟输入：

```text
12_最终归档/
├── 血压仪项目_最终归档_最终BOM_RevC_20260701_最终版.xlsx
├── 血压仪项目_最终归档_最终图纸包_RevC_20260701_最终版.pdf
├── 血压仪项目_最终归档_DFM关闭清单_RevC_20260701_已关闭.xlsx
├── 血压仪项目_最终归档_ECO关闭清单_RevC_20260701_已关闭.xlsx
└── 血压仪项目_最终归档_设计经验总结_RevA_20260701_最终版.docx
```

预期 Skill 行为：

1. 识别为 `归档专项`。
2. 检查最终 BOM、最终图纸、DFM/ECO 关闭清单、测试报告和问题关闭清单。
3. 若缺少测试验证报告，标记归档资料不完整。
4. 输出高频问题点、可复用经验、不建议复用的设计方案或风险点。
5. 标记是否写入最终归档为人工确认。

稳定性判断：通过。

## 仍需后续完善

这些不是第一版试用阻塞项，但进入正式系统前建议补齐：

1. 公司正式阶段名称和 Gate 标准。
2. 公司正式文件命名字典和资料类型字典。
3. 风险等级是否需要和 QMS、设计开发流程或 FMEA 严重度对齐。
4. ECO/ECN 审批角色、会签顺序和批准权限。
5. AI 报告是否可作为正式质量记录，还是只能作为辅助材料。
6. 敏感资料脱敏规则。
7. 文档解析失败时的最小人工补充要求。
8. DFM 问题是否需要增加公司自定义分类，例如材料、表面处理、包装、标签、供应商能力。
9. 导出 Excel、Word、PPT、PDF 的正式格式模板。
10. 规则更新建议的审批人和发布流程。

## 最终判断

`.cursor/skills/structural-project-review/` 当前规则稳定性满足第一阶段目标：可以用于 Cursor 内进行真实项目资料包的试用审查。

推荐下一步：

1. 选一个历史项目资料包做完整审查。
2. 记录 AI 的阶段判断、缺失资料、风险等级、DFM/ECO 提取结果是否符合工程师预期。
3. 把误判、漏判和不合适的风险等级记录为规则更新建议。
4. 连续验证 3 到 5 个项目后，再固化为 `structural-review-agent/` 的可运行模块逻辑。
